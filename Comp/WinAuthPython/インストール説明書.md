# WinAuthPython - インストール説明書

## システム概要

**IIS + YARP + Python** の構成で、Windows統合認証ユーザー名をPythonアプリに渡します。

```
Browser (社内ユーザー)
      ↓ Windows統合認証 (NTLM / Kerberos)
IIS
      ↓ ASP.NET Core in-process hosting
YARP リバースプロキシ (AuthProxy)
      - 認証済みユーザー名を X-Auth-User ヘッダーに付加
      ↓ HTTP (localhost)
Python server (waitress / uvicorn)
      ↓
Python Web App (Flask / FastAPI)
```

**ARR（Application Request Routing）との違い**:

ARRのURL RewriteはWindows認証完了前に実行されるため、`REMOTE_USER` を取得できません。
YARPはASP.NET Coreミドルウェアとして動作するため、認証後に `HttpContext.User.Identity.Name` からユーザー名を取得してヘッダーに付加できます。

## 1. システム要件

### 1.1 ハードウェア要件
- **プロセッサ**: 1GHz以上
- **メモリ**: 最小 2GB（推奨 4GB）
- **ディスク**: 最小 500MB

### 1.2 ソフトウェア要件

#### OS
- **Windows Server 2019 以上**（推奨：Windows Server 2022）
- または **Windows 10/11**

#### .NET 8.0
- .NET 8.0 SDK（開発時）
- .NET 8.0 Hosting Bundle（IISホスティング時）

#### IIS
- IIS 10.0 以上
- 必須機能:
  - Windows Authentication
  - ASP.NET Core Module v2（Hosting Bundle に含まれる）
  - ※ ARR モジュールは不要

#### Active Directory
- ドメイン環境（Windows認証用）

#### Python
- Python 3.9 以上
- waitress または uvicorn（本番用 WSGIサーバー）

## 2. インストール手順

### 2.1 前提条件

- .NET 8.0 SDK
- Python 3.9 以上

### 2.2 ステップ 1: .NET 8.0 SDKのインストール

1. https://dotnet.microsoft.com/download/dotnet/8.0 にアクセス
2. **"Download .NET SDK"** をクリック（Windows installer x64）
3. `dotnet-sdk-8.0.x-win-x64.exe` をダウンロードして実行
4. デフォルト設定でインストール

**確認**:
```powershell
dotnet --version
# 出力例: 8.0.x
```

### 2.3 ステップ 2: YARPプロジェクトの作成

```powershell
mkdir AuthProxy
cd AuthProxy
dotnet new web -n AuthProxy
cd AuthProxy
dotnet add package Yarp.ReverseProxy
```

**期待される出力**:
```
Build succeeded.
```

### 2.4 ステップ 3: Program.cs の実装

`Program.cs` を以下の内容に置き換え：

```csharp
using Microsoft.AspNetCore.Server.IISIntegration;
using Microsoft.AspNetCore.Authorization;

var builder = WebApplication.CreateBuilder(args);

// IISのWindows認証を利用
builder.Services.AddAuthentication(IISDefaults.AuthenticationScheme);
builder.Services.AddAuthorization(options =>
{
    options.FallbackPolicy = new AuthorizationPolicyBuilder()
        .RequireAuthenticatedUser()
        .Build();
});

// YARPリバースプロキシの設定
builder.Services.AddReverseProxy()
    .LoadFromConfig(builder.Configuration.GetSection("ReverseProxy"));

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();

// Windows認証ユーザー名を X-Auth-User ヘッダーに付加
app.Use(async (context, next) =>
{
    var userName = context.User.Identity?.Name ?? string.Empty;
    context.Request.Headers["X-Auth-User"] = userName;
    await next();
});

app.MapReverseProxy();

app.Run();
```

### 2.5 ステップ 4: 設定ファイルの更新

`appsettings.json` を以下の内容に変更：

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.AspNetCore": "Warning",
      "Yarp": "Warning"
    }
  },
  "AllowedHosts": "*",
  "ReverseProxy": {
    "Routes": {
      "route1": {
        "ClusterId": "pythonBackend",
        "Match": {
          "Path": "{**catch-all}"
        }
      }
    },
    "Clusters": {
      "pythonBackend": {
        "Destinations": {
          "destination1": {
            "Address": "http://localhost:8000/"
          }
        }
      }
    }
  }
}
```

**必須の修正項目**:
- `ReverseProxy.Clusters.pythonBackend.Destinations.destination1.Address`: Pythonサーバーのアドレス

### 2.6 ステップ 5: Pythonアプリの作成

例：Flask

```python
# app.py
from flask import Flask, request

app = Flask(__name__)

@app.route("/")
def index():
    user = request.headers.get("X-Auth-User", "unknown")
    return f"Hello, {user}"

if __name__ == "__main__":
    from waitress import serve
    serve(app, host="127.0.0.1", port=8000)
```

**必要なパッケージ**:
```
pip install flask waitress
```

**起動**:
```
python app.py
```

### 2.7 ステップ 6: 開発環境での実行確認

IIS を使わず `dotnet run`（Kestrel）で動作確認します。

> **注意**: Kestrel では `IISDefaults.AuthenticationScheme` が機能しません。開発時は `AddNegotiate()` を使う必要があります。

#### 1. Negotiate認証パッケージの追加

```powershell
cd AuthProxy
dotnet add package Microsoft.AspNetCore.Authentication.Negotiate
```

#### 2. Program.cs の修正（環境対応）

`Program.cs` の認証設定部分を以下のように変更：

```csharp
using Microsoft.AspNetCore.Server.IISIntegration;
using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.Authentication.Negotiate;

var builder = WebApplication.CreateBuilder(args);

// 開発時（Kestrel）は Negotiate、本番（IIS）は IISDefaults を使用
if (builder.Environment.IsDevelopment())
{
    builder.Services.AddAuthentication(NegotiateDefaults.AuthenticationScheme)
        .AddNegotiate();
}
else
{
    builder.Services.AddAuthentication(IISDefaults.AuthenticationScheme);
}

builder.Services.AddAuthorization(options =>
{
    options.FallbackPolicy = new AuthorizationPolicyBuilder()
        .RequireAuthenticatedUser()
        .Build();
});

builder.Services.AddReverseProxy()
    .LoadFromConfig(builder.Configuration.GetSection("ReverseProxy"));

var app = builder.Build();

app.UseAuthentication();
app.UseAuthorization();

app.Use(async (context, next) =>
{
    var userName = context.User.Identity?.Name ?? string.Empty;
    context.Request.Headers["X-Auth-User"] = userName;
    await next();
});

app.MapReverseProxy();

app.Run();
```

#### 3. Pythonアプリを起動

```powershell
python app.py
# → http://127.0.0.1:8000 で待ち受け開始
```

#### 4. YARPプロキシを開発モードで起動

別のターミナルで：

```powershell
cd AuthProxy
$env:ASPNETCORE_ENVIRONMENT = "Development"
dotnet run
```

**期待される出力**:
```
info: Microsoft.Hosting.Lifetime[14]
      Now listening on: http://localhost:5000
      Now listening on: https://localhost:7000
info: Microsoft.Hosting.Lifetime[0]
      Application started. Press Ctrl+C to exit.
```

#### 5. 動作確認

ブラウザで以下にアクセス：

```
http://hostname:5000/
```

**確認事項**:
- Windowsログインが自動認証されること（ログイン画面が出ない）
- Pythonアプリのレスポンスが返ること
- レスポンスに `DOMAIN\username` 形式のユーザー名が含まれること

> **Tip**: `http://localhost:5000` では Windows認証が失敗することがあります。マシン名 (`http://hostname:5000`) を使ってください。

## 3. IISでのホスティング

### 3.1 環境構築（IIS）

#### ステップ 1: IISのインストール

PowerShell（管理者）で実行：

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName IIS-WebServerRole

Enable-WindowsOptionalFeature -Online -FeatureName IIS-CommonHttpFeatures
Enable-WindowsOptionalFeature -Online -FeatureName IIS-StaticContent
Enable-WindowsOptionalFeature -Online -FeatureName IIS-DefaultDocument
Enable-WindowsOptionalFeature -Online -FeatureName IIS-HttpErrors
Enable-WindowsOptionalFeature -Online -FeatureName IIS-HealthAndDiagnostics
Enable-WindowsOptionalFeature -Online -FeatureName IIS-HttpLogging
Enable-WindowsOptionalFeature -Online -FeatureName IIS-Security
Enable-WindowsOptionalFeature -Online -FeatureName IIS-WindowsAuthentication
Enable-WindowsOptionalFeature -Online -FeatureName IIS-RequestFiltering
```

**確認**:
```powershell
iisreset
```

#### ステップ 2: .NET 8.0 Hosting Bundleのインストール

1. https://dotnet.microsoft.com/download/dotnet/8.0 にアクセス
2. **"Hosting Bundle"** をダウンロード（`dotnet-hosting-8.0.x-win.exe`）
3. 管理者として実行してインストール

> **重要**: Hosting Bundle に含まれる **ASP.NET Core Module v2** が、IIS と YARP アプリの連携を担います。ARR のような追加モジュールのインストールは不要です。

**確認**（PowerShell 再起動後）:
```powershell
dotnet --info
```

### 3.2 アプリケーションのデプロイ

#### ステップ 1: アプリケーションの公開

```powershell
cd AuthProxy
dotnet publish -c Release -o "C:\inetpub\wwwroot\authproxy"
```

**確認**:
```powershell
dir "C:\inetpub\wwwroot\authproxy"
# AuthProxy.dll が表示されることを確認
```

#### ステップ 2: ログディレクトリの作成

```powershell
New-Item -ItemType Directory -Path "C:\inetpub\wwwroot\authproxy\logs" -Force
```

#### ステップ 3: IIS マネージャーでサイトを作成

1. **IIS マネージャー**を起動
2. 左側ツリーから **"Sites"** を右クリック → **"Add Website"**
3. 以下を設定：

| 項目 | 値 |
|------|-----|
| Site name | AuthProxy |
| Application pool | AuthProxy（新規作成） |
| Physical path | C:\inetpub\wwwroot\authproxy |
| Binding Type | https |
| Port | 443 |
| SSL certificate | 有効な証明書 |

#### ステップ 4: アプリケーション プールの設定

1. **Application Pools** から **"AuthProxy"** を選択
2. 右側 **"Basic Settings"** を開く
3. 以下を設定：

| 項目 | 値 |
|------|-----|
| .NET CLR version | No Managed Code |
| Managed pipeline mode | Integrated |
| Identity | ApplicationPoolIdentity |

#### ステップ 5: Windows認証の有効化

1. **Sites > AuthProxy** を選択
2. 中央パネルから **"Authentication"** をダブルクリック
3. **Windows Authentication** を **Enable** に設定
4. **Anonymous Authentication** を **Disable** に設定

これにより、ブラウザのWindowsログインが自動認証されます（ログイン画面は表示されない）。

#### ステップ 6: ログディレクトリのアクセス権設定

PowerShell（管理者）で実行：

```powershell
$path = "C:\inetpub\wwwroot\authproxy\logs"
$user = "IIS AppPool\AuthProxy"

$acl = Get-Acl $path
$rule = New-Object System.Security.AccessControl.FileSystemAccessRule(
    $user,
    "Modify",
    "ContainerInherit,ObjectInherit",
    "None",
    "Allow"
)
$acl.AddAccessRule($rule)
Set-Acl -Path $path -AclObject $acl
```

#### ステップ 7: ファイアウォール設定

PowerShell（管理者）で実行：

```powershell
New-NetFirewallRule -DisplayName "IIS HTTPS" -Direction Inbound -Action Allow -Protocol TCP -LocalPort 443
```

#### ステップ 8: Pythonアプリの常時起動設定

タスクスケジューラまたは NSSM (Non-Sucking Service Manager) でPythonアプリをWindowsサービスとして登録する。

手動起動（確認用）：
```
python C:\apps\pythonapp\app.py
```

1. IIS マネージャーから **"AuthProxy"** サイトを右クリック → **"Start"**

## 4. トラブルシューティング

### 4.1 認証エラー（401 Unauthorized）

**解決方法**:
1. IISの認証設定を確認
   ```
   IIS Manager > Sites > AuthProxy > Authentication
   → Windows Authentication が Enable か確認
   → Anonymous Authentication が Disable か確認
   ```
2. クライアントマシンがドメインに参加しているか確認
3. `http://localhost` ではなく、ホスト名でアクセスしているか確認
   ```
   http://localhost → 失敗することがある
   http://hostname  → 推奨
   ```

### 4.2 502 Bad Gateway（Pythonへの接続失敗）

**解決方法**:
1. Pythonサーバーが起動しているか確認
   ```powershell
   netstat -an | findstr :8000
   ```
2. `appsettings.json` の `Address` を確認
3. Pythonサーバーが `127.0.0.1` (または `0.0.0.0`) で待ち受けているか確認

### 4.3 X-Auth-User ヘッダーが空

**症状**: Pythonアプリで `request.headers.get("X-Auth-User")` が空文字

**解決方法**:
1. IIS Windows Authentication が Enable か確認
2. Anonymous Authentication が Disable か確認
3. Hosting Bundle が正しくインストールされているか確認
   ```powershell
   Get-Module -ListAvailable | Where-Object { $_.Name -like "*AspNetCore*" }
   ```

### 4.4 HTTPS接続エラー

**症状**: "この接続は隠されていません"

**解決方法**:
1. SSL証明書が正しくインストールされているか確認
   ```
   IIS Manager > Server Certificates
   ```
2. バインディングでSSL証明書が選択されているか確認
   ```
   IIS Manager > Sites > AuthProxy > Bindings
   ```

### 4.5 パフォーマンスの問題

**症状**: 応答が遅い

**解決方法**:
1. アプリケーション プールのアイドルタイムアウトを設定
   ```
   IIS Manager > Application Pools > AuthProxy > Advanced Settings
   → Idle Time-out を 0 に設定（アイドル回収を無効化）
   ```
2. `appsettings.json` のログレベルを `Warning` に設定済みか確認

## 5. 運用手順

### 5.1 日常的な監視

IISログを確認：
```powershell
# 最新のIISログの末尾を表示
Get-ChildItem "C:\inetpub\logs\LogFiles\W3SVC1" | Sort-Object LastWriteTime -Descending | Select-Object -First 1 | Get-Content -Tail 50
```

### 5.2 アプリケーション更新（YARPプロキシ）

```powershell
# 1. 新しいコードをビルド
cd AuthProxy
dotnet publish -c Release -o "C:\inetpub\wwwroot\authproxy-new"

# 2. サイトを停止
iisreset /stop

# 3. ディレクトリを切り替え
Rename-Item "C:\inetpub\wwwroot\authproxy" "authproxy-old"
Rename-Item "C:\inetpub\wwwroot\authproxy-new" "authproxy"

# 4. サイトを起動
iisreset /start

# 5. 動作確認後、古いディレクトリを削除
Remove-Item "C:\inetpub\wwwroot\authproxy-old" -Recurse
```

### 5.3 転送先の変更（ダウンタイムなし）

YARPは `appsettings.json` の変更をホットリロードできます：

```powershell
# appsettings.json の Address を編集するだけで転送先が切り替わる（再起動不要）
notepad "C:\inetpub\wwwroot\authproxy\appsettings.json"
```

### 5.4 アプリケーション プールのリサイクル

```powershell
Restart-WebAppPool -Name "AuthProxy"
```
